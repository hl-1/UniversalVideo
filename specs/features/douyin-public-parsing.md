# 抖音公开解析与下载规格

## 目标

支持无需 Cookie 的抖音公开视频解析与下载，并在平台接口、签名参数或风控变化时提供明确失败信息。

本功能不读取浏览器 Cookie，不支持私密、登录后可见、地区受限或已失效视频，也不承诺绕过平台访问控制。

## 输入与输出

- 输入必须是 HTTP/HTTPS 抖音页面链接或分享链接。
- 分享链接规范化后再进入解析流程。
- 解析成功统一返回 `ParseResponse`：标题、封面、时长、作者、原网页地址和可选格式。
- 抖音公开接口成功时返回 `douyin-public` MP4 格式。
- 浏览器嗅探成功时返回 `sniff:<encoded-media-url>` 格式，并最多保留 8 条候选线路。

## 解析顺序

1. 优先使用无头浏览器打开公开页面，监听请求和响应中的 MP4/HLS 媒体资源。
2. 浏览器嗅探失败后，调用抖音公开 item info 接口和分享页数据获取视频信息。
3. 专用公开解析失败后，使用 yt-dlp 通用解析。
4. yt-dlp 失败后，继续尝试静态页面嗅探和浏览器嗅探。
5. 所有路径失败后返回 422，并将平台原始错误转换为简洁中文提示。

解析缓存按规范化页面 URL 保存，10 分钟后失效，用于避免重复打开页面并校验后续预览请求。

## 下载顺序

1. `sniffpair:` 使用 ffmpeg 合并已捕获的视频流和音频流。
2. `sniff:` 直接下载捕获的 MP4，或使用 ffmpeg 下载 HLS 并封装为 MP4。
3. 抖音链接优先重新调用公开 item info 获取无水印播放地址并流式下载。
4. 专用下载失败后回退 yt-dlp。

下载过程校验 HTTP 状态和内容类型，拒绝图片、空文件和非视频响应。临时文件使用 `.part` 后缀，成功后再替换为最终文件。

## 请求头与安全边界

- 抖音请求统一使用常规浏览器 User-Agent、`Origin: https://www.douyin.com` 和对应 Referer。
- 只处理公开媒体资源，不把 Cookie 写入配置、日志或响应。
- 错误信息不得暴露完整签名媒体地址。
- 文件名移除 Windows 非法字符并限制长度。
- 图片和媒体代理仅允许 HTTP/HTTPS 地址，并校验返回内容类型。

## 失败处理

- 无法提取视频 ID：返回链接无效或不受支持提示。
- 公开接口无 item 信息：提示视频可能失效、非公开或平台接口已调整。
- `fresh cookies`：说明系统已经尝试专用公开解析与 yt-dlp，不要求用户上传 Cookie。
- 加密参数或业务码变化：提示公开接口不可用并继续执行后续降级路径。
- 下载期间网络失败：删除临时文件并保留可读错误信息。

## 测试要求

- 覆盖链接规范化、视频 ID 提取和公开接口响应解析。
- 覆盖浏览器嗅探缓存、MP4/HLS 格式生成和降级顺序。
- 覆盖非视频内容、空文件、临时文件清理和 yt-dlp 回退。
- 与抖音总结页面文案兜底及在线预览的格式契约保持一致。
