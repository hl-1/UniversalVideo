# 最近输入链接功能规格

## 目标

在解析输入框获得焦点时展示用户最近提交的公开视频链接，便于重复解析和下载。

该功能只保存在当前浏览器本地，不上传服务器，不跨浏览器或设备同步。

## 保存规则

- 存储键为 `videodream.recentLinks`。
- 最多保存 5 条链接。
- 用户发起解析且输入非空时保存记录，不要求解析成功。
- 保存前移除链接首尾空白。
- 空字符串不保存。
- 重复链接移动到列表首位，不产生重复项。
- 列表按最近使用时间从新到旧排列。

## 读取与兼容

- 页面挂载后从 `localStorage` 读取。
- 只接受 JSON 数组中的字符串元素，其他元素丢弃。
- 非数组、损坏 JSON 或存储读取异常返回空列表。
- 历史数据同样执行去空、去重和 5 条上限归一化。
- `localStorage` 写入失败时保留当前内存列表，不阻塞解析请求。

## 前端交互

- 输入框使用 `autocomplete="off"`，由应用展示自己的历史列表。
- 输入框获得焦点且存在历史记录时显示下拉列表。
- 点击历史记录后填入输入框并关闭列表，不自动发起解析。
- 输入框失焦后短暂延迟关闭，确保鼠标点击记录可以先完成。
- 每条记录展示链接文本，超长文本必须截断或换行，不能撑破工作台。

## 隐私与安全

- 链接只存储在浏览器 `localStorage`。
- 后端接口和日志不新增历史记录字段。
- 不保存 Cookie、请求头、解析结果或媒体签名地址。
- 清理浏览器站点数据会删除历史记录。

## 测试要求

- 覆盖首尾空白清理和空输入忽略。
- 覆盖重复链接前移。
- 覆盖最多 5 条限制和顺序。
- 覆盖非法 JSON、非数组及非字符串历史元素。
- 覆盖存储读取和写入异常时的降级行为。
