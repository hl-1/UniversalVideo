# 抖音在线预览功能规格

## 目标

在解析结果卡片左侧封面区域提供抖音视频在线播放。解析完成后仍显示封面，用户点击中央播放按钮后，封面原位切换为浏览器原生视频播放器。

首版仅保证已解析出 MP4 线路的抖音公开视频，不支持 HLS 在线播放，也不改变下载和 AI 总结流程。

## 用户交互

1. 抖音解析成功且存在 MP4 格式时，封面中央显示播放按钮。
2. 用户点击播放按钮后，左侧区域进入“正在连接视频”状态。
3. 预览会话创建成功后，左侧原位显示带原生控件的 `<video>`，支持暂停、音量、进度拖动和全屏。
4. 播放器不自动播放，使用解析封面作为 `poster`，并启用 `playsinline`。
5. 创建会话、加载媒体或远端播放失败时，左侧恢复封面并显示可操作的失败提示。
6. 解析新链接或重新解析时，清理旧播放器和预览状态。
7. 预览状态不影响下载格式选择、下载任务和 AI 总结。

## 接口设计

### 创建预览会话

`POST /api/previews`

请求体：

```json
{
  "url": "https://www.douyin.com/video/123",
  "format_id": "sniff:..."
}
```

响应体：

```json
{
  "preview_url": "/api/previews/<token>/content",
  "expires_in": 600
}
```

约束：

- `url` 必须是抖音页面地址。
- `format_id` 必须是 `douyin-public` 或当前解析缓存中与该页面完全匹配的 `sniff:` MP4 格式。
- `sniffpair:`、HLS、非视频资源以及未关联到解析缓存的地址全部拒绝。
- 后端不向前端返回真实 CDN 地址。

### 获取视频内容

`GET /api/previews/{token}/content`

- 令牌为随机短期令牌，10 分钟后失效。
- 仅接受无 Range 请求或单个合法 `bytes` Range 请求。
- 后端使用抖音所需的 User-Agent、Origin 和 Referer 请求远端视频。
- 转发 `Content-Type`、`Content-Length`、`Content-Range`、`Accept-Ranges`，保留远端 `200` 或 `206` 状态。
- 使用流式响应，并在客户端断开或读取结束后关闭远端连接。

## 数据和安全边界

- 预览会话仅保存在进程内存中，记录媒体地址、来源页面、创建时间和过期时间。
- 创建会话时验证媒体协议为 HTTP/HTTPS，并验证媒体格式、抖音来源和解析缓存关联，防止接口成为任意 URL 代理。
- 过期会话在读取时删除；创建新会话时顺带清理过期记录。
- 预览地址失效后要求重新解析，不持久化 CDN 地址。
- 错误响应不包含完整远程签名地址或敏感查询参数。

## 前端状态

- `idle`：显示封面和播放按钮。
- `loading`：封面上显示连接状态并禁用重复操作。
- `ready`：显示原生视频播放器。
- `error`：恢复封面并显示失败提示，可重新点击或重新解析。

MP4 预览格式优先使用当前选中的 MP4；当前选择不是 MP4 时，使用解析结果中的第一个 MP4。没有 MP4 时不显示播放按钮。

## 失败处理

- 非抖音或无 MP4：前端不提供播放入口，接口调用则返回 422。
- 缓存不匹配或令牌无效：返回 404。
- 令牌过期：返回 410。
- 远端拒绝、超时或返回非视频内容：返回 502，并向用户显示简洁中文提示。
- 视频元素触发 `error`：销毁当前媒体地址并回到封面状态。

## 测试要求

- 后端单元测试覆盖合法会话、非抖音拒绝、缓存不匹配、HLS 拒绝、令牌无效与过期。
- 后端流式响应测试覆盖普通 `200`、Range `206`、关键响应头和远端错误。
- 前端测试覆盖 MP4 格式选择和无 MP4 情况。
- 运行后端测试、Python 编译检查、前端测试和生产构建。
- 使用浏览器验证封面、加载、播放和错误恢复布局在桌面及窄屏下无重叠。
